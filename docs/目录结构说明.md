## 📋 公开策略说明

### 公开原则

1. **核心代码公开**：源代码、测试代码、配置文件
2. **技术文档公开**：PRD（v1.2）、SDD（v1.1）、发布指南
3. **项目说明公开**：README、LICENSE、CHANGELOG

### 禁止公开原则

1. **敏感信息禁止**：Token、恢复代码、密码等
2. **构建产物禁止**：dist/、build/ 等自动生成文件
3. **历史草稿禁止**：prd_bak/ 等历史版本文档
4. **临时文件禁止**：缓存、日志等临时文件

## 🎯 快速参考

### 要添加新文件

```bash
# 1. 判断文件类型
# - 源代码/测试/文档 → 放在对应目录，自动可上传
# - 敏感信息 → 放入 docs/PyPI/ 或其他子目录
# - 构建产物 → 放在 dist/ 或 build/，自动忽略

# 2. 检查状态
git status

# 3. 提交
git add <文件>
git commit -m "说明"
```

### 要禁止敏感文件

```bash
# 方法 1：放入已配置的禁止目录
mv 敏感文件.md docs/PyPI/

# 方法 2：创建新的禁止目录
mkdir docs/private/
mv 私人文档.md docs/private/

# 方法 3：更新 .gitignore
echo "路径/到/目录/" >> .gitignore
```
